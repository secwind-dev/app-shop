คู่มือการใช้งาน Claude Code ใน VS Code

คู่มือนี้จะแนะนำ Trick และวิธีการใช้งาน Claude Code Extension ใน Visual Studio Code ให้มีประสิทธิภาพสูงสุด เพื่อเป็นผู้ช่วยในการเขียนโค้ดของคุณ

เริ่มต้นใช้งาน Claude Code

  • **การติดตั้ง:**
    • เปิด VS Code
    • ไปที่ Extensions (Ctrl+Shift+X หรือ Cmd+Shift+X)
    • ค้นหา "Claude Code" และติดตั้ง
  • **การล็อกอิน:**
    • หลังจากติดตั้ง Claude Code จะให้คุณเลือกล็อกอิน:
    • **1. Claude account with subscription:** เหมาะสำหรับผู้ใช้งานบ่อย มีราคาคงที่ (เช่น $20/เดือนสำหรับ Pro) สามารถใช้ Claude.ai ได้ด้วย
    • **2. Anthropic Console account:** คิดเงินตามการใช้งานจริง (API usage billing) เหมาะสำหรับใช้งานไม่บ่อย
    • หากเบราว์เซอร์ไม่เปิดอัตโนมัติ ให้คัดลอก URL ที่แสดงใน Terminal ไปวางในเบราว์เซอร์เพื่อทำการล็อกอิน และคัดลอกโค้ดที่ได้กลับมาวางใน VS Code
  • **การตั้งค่า Terminal (Terminal Setup):**
    • Claude Code จะถามว่า "Use Claude Code's terminal setup?"
    • **แนะนำ:** เลือก Yes, use recommended settings เพื่อประสบการณ์การเขียนโค้ดที่ดีที่สุด (เช่น Shift+Enter สำหรับขึ้นบรรทัดใหม่)
  • **การเรียกใช้งาน:**
    • **สำคัญ:** ควรเปิด VS Code โดยตรงจากโฟลเดอร์ Root ของโปรเจกต์ของคุณ (File > Open Folder...) เพื่อให้ Claude Code สามารถเข้าถึงและเข้าใจโครงสร้างโปรเจกต์ได้ทั้งหมด
    • Claude Code จะเปิด Terminal ขึ้นมาเอง พร้อมข้อความต้อนรับและคำแนะนำ

Trick & Best Practices ในการใช้งาน

1. การกำหนดบริบท (Context is King)

Claude Code ทำงานได้ดีที่สุดเมื่อมันเข้าใจบริบทของโค้ดที่คุณต้องการให้มันช่วยเหลือ:

  • **เปิดไฟล์ที่เกี่ยวข้อง:** ก่อนพิมพ์ Prompt ให้ Claude ช่วยเหลือ ตรวจสอบให้แน่ใจว่าไฟล์โค้ดที่คุณต้องการให้ Claude ทำงานด้วยถูกเปิดใน VS Code และเป็นไฟล์ Active (ที่กำลังแสดงผลอยู่) Claude จะใช้ข้อมูลจากไฟล์ที่เปิดอยู่เป็นหลัก
  • **เปิดโฟลเดอร์โปรเจกต์:** ควรเปิด VS Code โดยตรงที่ Root ของโฟลเดอร์โปรเจกต์ของคุณ (File > Open Folder...) การทำเช่นนี้ช่วยให้ Claude เข้าใจโครงสร้างโปรเจกต์, dependencies (จาก package.json), และไฟล์อื่นๆ ที่เกี่ยวข้อง
  • **เลือกโค้ดบล็อก:** สำหรับบางคำสั่ง (เช่น > explain, > refactor) คุณสามารถเลือก (Highlight) โค้ดบล็อกที่คุณต้องการให้ Claude โฟกัสก่อนที่จะพิมพ์คำสั่งได้ ทำให้ Claude ทำงานได้ตรงจุดมากขึ้น

2. Prompting ที่มีประสิทธิภาพ

การเขียน Prompt ที่ดีคือกุญแจสำคัญในการดึงศักยภาพของ Claude Code ออกมา:

  • **ชัดเจนและเฉพาะเจาะจง:**

    ไม่ดี: "แก้ไขบั๊กนี้"

    ดีกว่า: "บั๊กนี้เกิดขึ้นเมื่อผู้ใช้ล็อกอินด้วยข้อมูลที่ไม่ถูกต้อง และคาดว่าจะแสดงข้อความ error แต่กลับไม่มีอะไรเกิดขึ้น โปรดตรวจสอบฟังก์ชัน handleLogin ในไฟล์ auth.ts เพื่อหาและแก้ไขปัญหานี้"

  • **ระบุ Output ที่คาดหวัง:** คุณต้องการโค้ดที่แก้ไขแล้ว? คำอธิบาย? หรือชุดของขั้นตอนในการแก้ไข? ระบุให้ชัดเจน
  • **ให้ตัวอย่าง (ถ้าเป็นไปได้):** ถ้ามีโค้ดตัวอย่างที่คล้ายกับสิ่งที่คุณต้องการ หรือตัวอย่างของ Input/Output ที่คาดหวัง จะช่วยให้ Claude เข้าใจได้ดีขึ้น
  • **แบ่งเป็นขั้นตอนเล็กๆ:** สำหรับงานที่ซับซ้อน แทนที่จะขอให้ทำทั้งหมดในครั้งเดียว ลองแบ่งเป็น Prompt เล็กๆ เช่น "ขั้นแรก สร้างโครงสร้างไฟล์สำหรับ feature X" จากนั้น "ขั้นที่สอง เขียนโค้ดสำหรับ component Y ในไฟล์นั้น"
  • **ใช้คำสั่งนำหน้า (Prefix Commands):** Claude Code มีคำสั่งนำหน้าที่ช่วยกำหนดประเภทของงานใน Terminal (พิมพ์แล้วกด Space):
    • > try "...": ลองแก้ไข/ทำบางอย่าง
    • > explain "...": อธิบายโค้ดหรือแนวคิด
    • > generate "...": สร้างโค้ดใหม่
    • > refactor "...": ปรับปรุงโค้ด
    • > debug "...": ช่วยดีบักปัญหา
    • > ask "...": ถามคำถามทั่วไปเกี่ยวกับโค้ด/แนวคิด
    • ?: พิมพ์ ? แล้วกด Enter เพื่อดู Shortcut และคำสั่งเพิ่มเติม
เคล็ดลับ: การพิมพ์ Prompt เป็นภาษาอังกฤษมักจะได้ผลลัพธ์ที่ดีกว่า เนื่องจากโมเดล AI ถูกเทรนมาด้วยข้อมูลภาษาอังกฤษจำนวนมาก

3. การตรวจสอบและยอมรับ/ปฏิเสธการแก้ไข

  • **ตรวจสอบเสมอ:** AI ไม่ได้สมบูรณ์แบบเสมอไป ตรวจสอบโค้ดที่ Claude Code แนะนำอย่างละเอียดก่อนที่จะยอมรับ
  • **ใช้ Diff View:** เมื่อ Claude Code เสนอการเปลี่ยนแปลงโค้ด มันมักจะแสดง Diff View ที่ให้คุณเห็นความแตกต่างระหว่างโค้ดเดิมและโค้ดที่เสนออย่างชัดเจน ใช้ประโยชน์จากสิ่งนี้เพื่อตรวจสอบ
  • **ยอมรับ/ปฏิเสธอย่างระมัดระวัง:** VS Code มี UI ที่ช่วยให้คุณยอมรับ (Accept) หรือปฏิเสธ (Reject) การเปลี่ยนแปลงที่ Claude Code เสนอได้ง่าย ใช้สิ่งนี้อย่างมีสติ
  • **ปรับแต่งก่อนยอมรับ:** บางครั้ง Claude Code อาจจะทำได้เกือบสมบูรณ์แบบ คุณสามารถแก้ไขเพิ่มเติมเองเล็กน้อยใน Diff View ก่อนที่จะยอมรับการเปลี่ยนแปลง

4. การจัดการกับปัญหาทั่วไป

  • **ถ้า Claude "ติดลูป" หรือให้คำตอบที่ไม่เกี่ยวข้อง:**
    • ลองกด Ctrl+C (หรือ Cmd+C บน Mac) ใน Terminal เพื่อยกเลิกการประมวลผลปัจจุบัน
    • ลอง Clear Terminal แล้วเริ่มต้น Prompt ใหม่
    • ปรับปรุง Prompt ของคุณให้ชัดเจนและเฉพาะเจาะจงยิ่งขึ้น
  • **ตรวจสอบการเชื่อมต่อ:** ตรวจสอบให้แน่ใจว่าคุณยังคงเชื่อมต่อกับบัญชี Claude ของคุณอยู่ (บางครั้งอาจมีการหลุดการเชื่อมต่อ) ถ้าหลุดการเชื่อมต่อ ให้ล็อกอินใหม่
  • **อัปเดต Extension:** ตรวจสอบให้แน่ใจว่า Claude Code Extension ของคุณเป็นเวอร์ชันล่าสุดเสมอ เพื่อให้ได้ฟีเจอร์และ Bug Fixes ล่าสุด

5. การเรียนรู้และปรับตัว

  • **สังเกตพฤติกรรม:** ใช้เวลาสังเกตว่า Claude Code ตอบสนองต่อ Prompt รูปแบบต่างๆ อย่างไร สิ่งนี้จะช่วยให้คุณปรับแต่งสไตล์การ Prompt ของคุณให้มีประสิทธิภาพมากขึ้น
  • **ลองผิดลองถูก:** อย่ากลัวที่จะลอง Prompt ที่แตกต่างกัน หรือลองให้ Claude Code ทำงานที่หลากหลาย คุณจะเรียนรู้ข้อจำกัดและจุดแข็งของมันได้จากการทดลอง
สำคัญ: แม้ Claude Code จะเป็นผู้ช่วยที่ยอดเยี่ยม แต่ก็ไม่ควรพึ่งพา 100% ควรตรวจสอบโค้ดที่ AI สร้างขึ้นเสมอ เพื่อความถูกต้อง ความปลอดภัย และประสิทธิภาพ
